## The Problem

You're running AI agents on a remote server вҖ” Claude, GPT, custom harnesses, autonomous coding tools. They generate files, write reports, draft documents. But:

- **You can't see the output** without SSH-ing in and running `cat`
- **You can't give feedback** on line 42 of a draft without copy-pasting into a chat
- **Your client can't access** the server at all вҖ” you end up emailing files back and forth

## The Solution

Asyre File runs on the same server as your AI agents. It gives you a **browser UI** to view, edit, annotate, and share every file the agent touches вҖ” and a **REST API** for agents to read and write programmatically.

## How It Works

<p align="center">
<img src="docs/assets/dark-mode.png" width="80%" alt="Asyre File">

### 1. AI writes a file вҶ’ You see it instantly

Your agent creates or updates files via the REST API. Open the browser вҖ” the file tree updates in real-time.



### 2. You annotate specific lines вҶ’ Copy to your AI

Select lines, write feedback like "too formal, rewrite conversationally". Click **Copy** вҖ” the annotation includes the **full server path** so any AI can find the file.

![Annotate lines](docs/assets/annotate-lines.png)

**What gets copied** вҖ” full server paths so AI can locate the files:

![Annotation copy](docs/assets/annotate-copy.png)

Paste this into Claude, GPT, or any AI вҖ” it knows exactly which file and which lines to fix.

### 3. Batch annotate multiple files

Select files with **Cmd+Click**, hit **Annotate**, write your feedback, then **Copy** or **Save**.

<p align="center">
<img src="docs/assets/annotate-batch-a.png" width="48%" alt="Batch Annotate">
<img src="docs/assets/annotate-batch.png" width="48%" alt="Batch Annotate Result">
</p>

### 4. Share with anyone вҖ” no account needed

Right-click a file вҶ’ **Copy link**. Or share an entire folder. Recipients get a clean read-only (or editable) view.

![Share link](docs/assets/share-link.png)

## Quick Start

### Option 1: Git Clone

```bash
git clone https://github.com/yzha0302/asyre-file.git
cd asyre-file
python3 server.py
```

Visit `http://localhost:8765` вҖ” the setup wizard creates your admin account.

### Option 2: Docker

```bash
git clone https://github.com/yzha0302/asyre-file.git
cd asyre-file
docker compose up -d
```

### Option 3: One-Line Install

```bash
curl -fsSL https://raw.githubusercontent.com/yzha0302/asyre-file/main/install.sh | bash
```

## Agent API

Give your AI agent an API token during setup, then it can read and write files:

```bash
# Write a file
curl -X PUT -H "Authorization: Bearer asf_your_token" \
  -H "Content-Type: application/json" \
  -d '{"content": "# Weekly Report\n\nGenerated by agent."}' \
  http://your-server:8765/api/v1/files/reports/week-14.md

# Read a file
curl -H "Authorization: Bearer asf_your_token" \
  http://your-server:8765/api/v1/files/reports/week-14.md

# List all files
curl -H "Authorization: Bearer asf_your_token" \
  http://your-server:8765/api/v1/files

# Search
curl -H "Authorization: Bearer asf_your_token" \
  http://your-server:8765/api/v1/search?q=quarterly

# Delete (moves to trash)
curl -X DELETE -H "Authorization: Bearer asf_your_token" \
  http://your-server:8765/api/v1/files/drafts/old.md
```

Tokens support granular permissions: `read`, `write`, `delete`.

## Features

### Editor
- Live Markdown editor with **20+ language modes** (JS, Python, HTML, JSON, YAML, etc.)
- Real-time preview with **Mermaid diagrams**, code highlighting, math
- **Dark & light themes** вҖ” everything adapts, including Mermaid and CodeMirror

<img src="docs/assets/light-mode.png" width="48%" alt="Light Mode">

### File Management
- **File tree** with colored icons per file type
- **Drag-and-drop** to move files between folders
- **Upload** via button, drag-and-drop onto page, or right-click "Upload here"
- **Right-click menu**: Open, Rename, Copy path, Copy link, Download, Select, Delete
- **Multi-select** with Cmd/Ctrl+Click вҶ’ batch Share, Annotate, Copy, Delete
- **Search** across all files
- **Trash & restore** with admin-only empty



### Collaboration
- **Multi-user auth** вҖ” Admin / Editor / Viewer roles
- **Path-based permissions** вҖ” scope each user to specific folders
- **Share links** вҖ” read-only or editable, single file or entire folder
- **PDF & Word export** with custom signatures and themes (optional: `pip install weasyprint python-docx`)

<p align="center">

### For AI Workflows
- **Annotations** вҖ” select lines, write feedback, copy with full server paths
- **Batch file annotations** вҖ” annotate multiple files at once
- **REST API** with Bearer token auth for agents
- **Activity log** вҖ” track all file changes (admin panel)
- **Saved annotations** вҖ” persist to server, agents can read via API
